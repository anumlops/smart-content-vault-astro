# Smart Content Vault — AGENTS.md

## Stack

- Astro v6 **SSR** (`output: 'server'`) deployed on Vercel via `@astrojs/vercel`
- Tailwind CSS v4 via `@tailwindcss/vite` (no PostCSS/tailwind config files)
- Prisma v5 + PostgreSQL (Neon)
- Session auth via httpOnly cookies (custom SHA-256 + base64 tokens; `better-auth` dep is unused)
- **Node >=22.12.0** (`.nvmrc` says 20, but `package.json` engines wins)

## Commands

| Command | Action |
|---|---|
| `npm run dev` | Astro dev server |
| `npm run build` | Astro production build |
| `npm run prisma:generate` | Generate Prisma client |
| `npm run prisma:push` | Push schema to DB |
| `npm run prisma:studio` | Prisma Studio GUI |
| `npm run setup` | Full first-time setup (install + generate + push) |

No test, lint, or typecheck scripts exist. `postinstall` auto-runs `prisma generate`.

## Architecture

### Content pipeline (two-pass)
1. **Save time** (`lib/processing.ts`): fetch HTML, regex-extract OG metadata, keyword-categorize (20 cats), generate tags → write to DB
2. **Post-save** (`lib/enrich.ts` → `modules/website/`): crawler (linkedom + Readability) → LLM analysis via OpenRouter (falls back: GPT-4o-mini → DeepSeek → Kimi → keyword) — stores in `aiSummary`, `aiTags`, `aiCategory`, `keyTakeaways`

### Two category systems
- `lib/categorizer.ts` — 20 categories, used at save time (stored in `SavedContent.category`)
- `modules/shared/constants.ts` — 10 categories, used by LLM enrichment (stored in `aiCategory`)
- Keep both in sync if adding categories

### Content processors
`src/modules/` implements `ContentProcessor` interface (`shared/types.ts`). Only `website/` is implemented; `youtube/` and `instagram/` are placeholders.

### Microservices (independent, not connected to the main app)
- `microservices/crawl4ai/` — Python web crawler
- `microservices/content-extractor/` — Python
- `microservices/website-intelligence-service/` — Node

### API patterns
- All endpoints verify session via `cookies.get('session')` → `verifySession()`
- `POST /api/content` accepts `formData`, `POST /api/content/quick` accepts JSON
- `POST /api/enrich` triggers LLM enrichment on-demand (accepts `{contentId}` or `{url}`)
- `/dev` endpoint is a browser-based enrichment test console

### Auth
- Password hashing: SHA-256(`password` + `AUTH_SECRET`)
- Session: base64(`${userId}:${timestamp}:${secret}`) — **not cryptographically secure**, just obfuscated
- Cookie: `session`, httpOnly, 7-day expiry

## Gotchas

- `dashboard.astro` queries ALL content without user filtering (bug — no auth check on the page route)
- `.env` has **committed secrets** — never push changes without checking
- `middleware/` directory exists but is empty
- `src/generated/prisma/` is gitignored (generated by `prisma generate`)
- `public/assets/placeholders/` referenced by `thumbnail.ts` — verify SVGs exist before adding new content types
